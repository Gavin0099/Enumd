## 緣由
在面對遇到客戶USB 問題時，常常都要透過錄製 usb 訊號或是 i2c 訊號來分析問題點，如果透過人工的方式來抓取的話，會耗費很多人力，如果用自動化測試的方式來抓取對應的log ，讓 Tool 反覆測試來找出問題點，會是一種比較好的測試方式
目前想法是透過Python 來串每個Tool ，目前找起來 i2c & Usb Trace Tool 都有提供 python api ，至於 bus hound 可能要透過 winauto 的方式來實作功能。
###  Pythone 架構
透過Main來決定要開啟那些功能，透過簡單的參數設定就可以讓使用者決定要錄製那些log
