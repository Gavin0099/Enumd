## 202510

- **已完成**

  - 為了實現跨平台UI的功能，所以修改了GL SDK的功能。

- **進行中**

  - HP OCI Tool
  
    - OCI Tool 採用三層式架構 (UI→DLL→底層)，底層的修改容易引發上層 UI 的潛在錯誤，先以導入版本交握機制為初期目標。
    
    - 同步建立並完善 OCI Tool 的版本修改歷史記錄，讓開發團隊可以更清楚維護脈絡與開發基礎。
  
  - 提供GL9511 PD Power Monitor 讓客戶可以驗證PD 的功能性。
  
  - Lenovo POC AI Monitor
  
  - Lenovo POC新案，預計最近幾個月要demo ，在開發時遇到很多HW問題，目前正積極與代工廠及韌體 (FW) 團隊協作，共同解決開發過程中遇到的關鍵硬體問題。
  
  - HP Mac OSDManager
  
    - 繼 Windows 版本後，為 HP 開發 Mac 版本的 OSDManager 工具，透過研究與導入 C# 跨平台框架，成功實現了大部分程式碼的共用。此方案大幅提升了開發效率，目前實作中。

- **Chance for better **

  - 在面對HW問題時，如何可以更好或更快的回覆其問題點，這是需要經驗傳承的。
  
  - 當面對前所未見的技術難題時，如何系統性地找出最佳解決方案?
  
  - 嘗試導入規格先行 (Spec-First)的作法，在專案初期便確立詳盡的規格文件，並以此作為開發的唯一依據，確保最終功能能完全符合預期。

## 202511

- **進行中**

  - Mac OSDManager
  
  - 目前功能已大致開發完成。因為在Mac 上導入C# 跨平台框架，開發過程中所遇到的 OS 相關問題，均已在相關人員支援下即時解決。
  
  - HP OCI Tool 支援多重 OS 需求討論
  
    - 為滿足 HP 明年新機型對多樣 OS 的支援(Win & Mac & Linux)，已完成需求討論。HP 決議將擴展韌體更新支援能力，並將此列為未來所有新專案的強制標準。目前三大平台大致符合要求，未來所需的開發資源會在高階機種所需的 Azure RTOS。
  
  - Lenovo POC 專案
  
    - 目前 Hub & Scaler 已可正常更新。已經先包裝對應的 Tool 給客戶驗證，PD 部分的更新功能，尚待客戶提供相應 Tool。目前責任歸屬明確，待取得 PD Update Tool 後即可進行最後 Release。
  
  - HP ISP Tool
  
    - 為導入 Tri Code ，已透過 C# 重新開發 UI 介面。新 UI 可直接呼叫 Tri Code function，並完整複刻原本UI，確保客戶端無需重新適應。
  
  - Etoken system
  
    - 撰寫相關技術文件。
    
    - 移除 Client 端的 MySQL 依賴，已經把此功能轉到Server端
    
    - 導入加密機制，避免傳送過程封包攔截導致的資料外洩

- **Chance for better **
