# Log 分類

## 不必要的 Log
在開發過程中，有時會產生一些不必要的 log 訊息。這些 log 可能包含一些敏感資訊或過於冗長的詳細資訊,對於分析問題並不太有幫助。因此,我們應該盡量避免產生這些不必要的 log。

## 更好懂的 Log
相反地,我們應該努力產生更有意義、更容易理解的 log 訊息。這些 log 應該能夠清楚地描述發生的事件,並提供足夠的上下文資訊,以便於後續的分析和問題排查。

## 不知道誰呼叫的
以下是一個示例 log 訊息,其中包含了一些不知道誰呼叫的資訊:

```c++
::DbgPrint(XD_INFO, "--\tDynamic:\n");
::DumpBuffer(XD_INFO, (uint8_t *)hubInfo.StringDesc[1], GL_STRING_DESCRIPTOR_LENGTH_MAX / 2);
2024-03-27 09:49:08 [info ] 000: 43323330 30303033 30640000 00000000
2024-03-27 09:49:08 [info ] 010: 00000000 00000000 00000000 00000000
```

這個 log 訊息包含了一些動態資訊,但沒有明確說明是由誰呼叫的。為了更好地理解這個 log 訊息,我們應該在程式碼中加入適當的註解,說明這些資訊的來源和用途。

總的來說,我們應該努力產生更有意義、更容易理解的 log 訊息,並在程式碼中加入適當的註解,以便於後續的分析和問題排查。
